Amazon S3 ・ ストレージ入門

バケットは「無限に伸びる箱」

容量を気にせず、置いた分だけ払う。サーバー不要のオブジェクトストレージ。

S3 は容量無制限のファイル置き場。サーバーを立てずに、写真・動画・バックアップを置けて、置いた分だけ課金。1ファイル最大 5TB まで。

① 📦

容量無制限

事前のサイズ確保が不要。増えても勝手に伸びる。

② 💰

使った分だけ

置いた容量とアクセス回数で課金。初期費用ゼロ。

③ 🌐

どこからでも

URL でアクセス。静的サイトの公開もできる。

01

そもそも「オブジェクトストレージ」って？ object storage

フォルダ階層ではなく、ファイル(=オブジェクト)をフラットな箱に放り込むタイプの保存方式。一つひとつにキー(名前)が付き、それで取り出す。

「フォルダ」に見えるのは表示上の演出。実体は名前(キー)だけ！
02

使い始めるまでの流れ

  1. 1バケットを作る
    世界で一意の名前を付ける。リージョンを選ぶ。
  2. 2ファイルを置く
    ドラッグ&ドロップ or CLI で aws s3 cp。
  3. 3アクセス権を決める
    既定は非公開。公開したい時だけポリシーで開ける。

🟢 向いてる

画像・動画・ログ・バックアップ・静的サイト配信

🔴 向かない

頻繁に一部だけ書き換えるDB用途、低レイテンシな読み書き
1 オブジェクトの最大サイズ: 5 TB / 1 回の PUT: 最大 5 GB（それ以上はマルチパートアップロード）
ストレージクラス取り出し相対コスト
S3 Standard即時高
S3 Glacier Flexible Retrieval数分〜数時間低
S3 Glacier Deep Archive最大 12 時間最安
⚠️ つまずきポイント バケット名は全世界で一意。よくある名前は既に取られていて作れない。
💡 受講生へのひとこと まずは1つバケットを作って画像を1枚置いてみよう。「公開URLで開く」まで触ると一気に理解が進む。
03

ストレージクラスの選び方

よく使うなら Standard。滅多に出さない保管は Glacier 系で大幅節約。
