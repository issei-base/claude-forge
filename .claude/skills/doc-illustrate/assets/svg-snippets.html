SVG 部品カタログ

コピーするのは <svg> 本体だけ。色は CSS 変数、座標は数十px刻みで調整。

0. 矢印 marker（全図の共通 defs）

フロー図に矢印を引くなら、図の冒頭に 1 回だけこの <defs> を置く。marker-end="url(#arrow)" で線の先に付く。

<defs>
  <marker id="arrow" viewBox="0 0 10 10" refX="8" refY="5"
          markerWidth="7" markerHeight="7" orient="auto-start-reverse">
    <path d="M0 0 L10 5 L0 10 z" fill="var(--accent-deep)"/>
  </marker>
</defs>

1. 横フロー（A → B → C）

リクエストの流れ・処理パイプライン（client → service → DB 等）。箱を増やすなら x を +180 ずつずらす。

2. アイコン付きボックス（単体パーツ）

サービス 1 つを表す基本ブロック。タイトル + 補足 1 行。これを並べて図にする。

3. 入れ子（外箱の中に内箱・親子関係）

VPC→Subnet→EC2、ロードバランサー→リスナー→ルール のような包含関係。外箱は accent-deep 枠で。

4. 分岐（1 → 多 / ルーティング）

1 つの入口から条件で複数先へ振り分け（ALB のルール→ターゲットグループ、SNS ファンアウト等）。曲線は C で。

5. グループ箱（中に複数ターゲット＋状態ドット）

ターゲットグループ・Auto Scaling グループなど「容器＋中身＋状態」。緑ドット＝正常。点線枠＝共有/任意。

6. 階層スタック（レイヤー構造）

OSI 層・ネットワーク階層・抽象度の積み重ね。下から上、または上から下に積む。

― 足りない形（天秤・ゲージ・円グラフ等）は references/visual-style.md の「図解の発想」を参照。凝った絵が要るなら無理に SVG にせず .features / .compare で代替してよい。
