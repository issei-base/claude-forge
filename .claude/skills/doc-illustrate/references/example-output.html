Claude ・ 新モデル発表（2026/05/28）

Opus 4.8 は「同じ値段で、もっと頼れる相棒」

Opus 4.7 をベースに、ベンチマーク全体を底上げ。判断力が上がり、価格は据え置きで今日から使える。

Claude Opus を新バージョン Opus 4.8 に更新。原典いわく 「Opus 4.7 をベースにベンチマーク全体を改善し、より効果的な協働相手になった。価格据え置きで今日から利用可能」。

特に効くのが エージェント作業での判断力。自分の書いたコードの欠陥を見逃す確率が 約 1/4（4倍 見逃しにくい）に。

① 🧐

判断力が上がった

ミスを拾い、筋の悪い計画に疑問を投げる。コードの欠陥見逃しが約4倍 起きにくく。

② ⚡

効率が良くなった

同じ賢さをより少ない手数で。ツール呼び出しが効率化、マルチモーダルは 61% 安いトークンコスト。

③ 🏷️

価格は据え置き

通常利用は Opus 4.7 と同じ $5 / $25（入力 / 出力・100万トークンあたり）。

01

「効く相棒」になった a more effective collaborator

一番の変化は エージェント作業での判断の確かさ。ミスを catch し、不確かなところは旗を立て、根拠の薄い断定を減らす——「ただ従う」のではなく「考えて返す」方向の改善。

Opus 4.8 is around four times less likely than its predecessor to allow flaws in code it has written to pass unremarked.
＝ 自分の書いたコードの「やらかし」をスルーする率が、Opus 4.7 比でおよそ 1/4 に。

04

大規模作業を「分担」して回す Dynamic Workflows

Claude が作業を計画し、1セッション内で数百の並列サブエージェントを走らせられる。Claude Code + Opus 4.8 なら、何十万行規模のコードベース移行をキックオフからマージまで一気通貫で。

Claude can plan the work and then run hundreds of parallel subagents in a single session ... Claude Code with Opus 4.8 can now carry out codebase-scale migrations across hundreds of thousands of lines of code from kickoff to merge.
05

開発者向け：価格と API for developers

呼び出しは claude-opus-4-8。価格は通常利用が Opus 4.7 据え置き、より速い fast mode は別建て。いずれも 100 万トークンあたり。

💡 API のうれしい変更 Messages API が messages 配列の中に system エントリを受け付けるように。プロンプトキャッシュを壊さずに、タスク途中で Claude への指示を更新できる。
06

安全性・アラインメント alignment

リリース前にアラインメント評価を実施。prosocial（向社会的）な指標で新たな高水準に達し、不整合な振る舞いの率は Opus 4.7 より大きく低下、としている。

reaches new highs on our measures of prosocial traits like supporting user autonomy and acting in the user's best interest ... rates of misaligned behavior (such as deception or cooperation with misuse) that are substantially lower than Opus 4.7.
