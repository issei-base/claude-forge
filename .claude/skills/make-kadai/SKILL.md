# make-kadai

受講生カリキュラムシートの**「次回までの宿題（課題欄）」に、テーマ指定のハンズオン課題を自動生成・自動書き込み**するスキル。SAMURAI ENGINEER のカリキュラムシート（レッスン回ごとに課題列が並ぶ構造）を対象にする。

役割分担:
- **どのセルに書くか / 実際の read・write** = `scripts/sheet_kadai.py`（決定的・テスト済み）。目視で列を数えると事故るので固定。
- **課題文そのものの生成** = このワークフロー（モデル）。直近に埋まった課題を**テンプレート**として書式を完全に踏襲し、テーマだけ差し替える。

同梱物:
- `scripts/sheet_kadai.py` — URL から spreadsheet_id+gid を割り出し、`N回目レッスン` ヘッダ行と `次回までの宿題` 課題行を引き当て、対象セルの特定・読み・書きを行う。`read` / `write` / `auth-check` の3コマンド。
- `references/sheet-structure.md` — 対象シートの構造とセル特定ロジックの前提。挙動が想定と違う時に読む。

## 🚫 守るルール

- **空きセルにだけ書く**。既に課題が入っているセルは `--overwrite` を付けない限り書かない（スクリプトがガードする）。上書きが要るなら必ずユーザーに確認してから。
- **AWS の手順は記憶で書かない**。コマンド・サービス名・上限・前提は [[aws-docs]] / [[aws-advisor]] skill で一次ソースに当ててから課題に落とす。
- **書き込み前に、生成した課題文と対象セル（例: `G375`）をユーザーに見せて確認**してから `write` する。「自動書き込み」でも、何をどこに入れるかは一度見せる。
- **書き込んだら必ず読み戻して問題が無いかレビューしてから報告する**（ステップ5）。書きっぱなしにしない。`verified: false` なら直す。
- main の課題列構造を壊さない。書くのは1セルだけ。

## 前提（初回だけ・認証）

Sheets API は **`gws`（Google Workspace CLI）経由**で叩く。gws は独自の検証済み OAuth クライアントを使うので、gcloud 既定クライアントの `spreadsheets` スコープ・ブロック（"このアプリはブロックされます"）を回避できる。**未認証/失効なら `read`/`write` が止まり、再認証コマンドを出す**ので、それをユーザーに実行してもらう:

```sh
gws auth login
```

（ブラウザが開く → **対象シートを編集できる Google アカウント**で承認）。`python3 scripts/sheet_kadai.py auth-check` が `{"auth":"ok"}` を返せば準備完了。

> gws 未導入なら `npm install -g @googleworkspace/cli`。`gws auth status` が `auth_method: oauth2` で `sheets.googleapis.com` を含んでいれば OK。gcloud ADC は既定クライアントの scope ブロックで使えないことがあるため、この skill は gws を既定の経路にしている。

## ワークフロー

### 1. 入力を確定する
- **スプレッドシート URL**（`gid=` 付き＝対象タブを含むもの）と **テーマ**（例: ALB, CloudFront, S3）を受け取る。どちらか欠けていれば聞く。
- 対象レッスンは原則 **`read` が返す「最初の空き課題セル」**。ユーザーが「6回目に」等と明示したら `--lesson 6`。

### 2. シートを読む（テンプレと対象セルを得る）
```sh
python3 scripts/sheet_kadai.py read --url "<URL>" [--lesson N]
```
返る JSON の見どころ:
- `target.cell`（書き込み先・例 `G5`）と `target.lesson` / `target.empty`。
- `template_lesson` / `template_text` … **直近に埋まった課題＝書式の手本**。これを骨格として踏襲する。
- `filled_lessons` … 既に出ている課題（積み上げの文脈。例: EC2→AMI→Auto Scaling と来ていれば、その続きとして設計する）。
- `hints.plan_for_this_lesson` … 直前回に記入された「次回レッスンの内容」＝この回で扱う予定。**テーマ未指定ならこれを主題のヒントに**（ユーザーがテーマを明示したらそちらが優先）。
- `hints.prev_taught` … 直前回の指導内容。自然な次ステップを選ぶ材料。

> 課題行（「次回までの宿題」）が複数ある月割りシートでは、`read` は**先頭ブロック**を採用する。別ブロックに書きたい時は `read` の出力で `target.cell` を確認し、`write --cell <A1>` で明示する。

### 3. 課題文を作る（テンプレ踏襲＋テーマ）
- `template_text` の**構成をそのまま真似る**。SAMURAI の手本書式は概ね:
  ```
  ハンズオン課題：<テーマ>の<動詞>
  【詳細な要件】
  <1〜2文で到達点>
  【具体的な手順と学習ポイント】
  1. … / 2. … / …（番号付きステップ）
  N. 動作確認とクリーンアップ（重要：確認後は作成物を削除/停止）
  ```
- **直前のレッスンの成果物に積み上げる**設計にする（例: ALB なら「Auto Scaling グループの前段に ALB を置く」）。`filled_lessons` と `hints` の流れから自然な接続を選ぶ。
- **AWS の正確さ**: サービス名・操作手順・前提（リスナー/ターゲットグループ/ヘルスチェック等）は [[aws-docs]] / [[aws-advisor]] で裏取りしてから書く。無料枠・課金の注意とクリーンアップ手順は必ず入れる（受講生の事故防止）。
- **参考リンクは実在する URL のみ**。記憶が不確かなら載せない（捏造禁止）。AWS は公式 docs（`https://docs.aws.amazon.com/ja_jp/...`）を優先。
- **受講生プロフィールを反映**する（テンプレや既存課題から読み取れる前提知識・進度。例: 基礎資格取得済なら用語説明は省きすぐ実践レベルで）。
- 文体・密度・絵文字の有無はテンプレに合わせる。**新しい列だけを足し、既存課題には触れない**。

### 4. 確認 → 書き込む
- 生成した課題文と `target.cell` をユーザーに提示する。
- 課題文を一時ファイルに書き、`write` する:
```sh
python3 scripts/sheet_kadai.py write --url "<URL>" --lesson N --value-file /tmp/kadai.txt
```
- `--lesson N`（または `--cell A1`）でスクリプトが**書き込み直前にセルを再解決**し、空きガードを通してから `values.update`(USER_ENTERED) する。

### 5. 書き込み後に必ず確認・レビューする（必須）
書きっぱなしにしない。書き込み後は**必ず**自分で読み戻して問題が無いか確かめてから報告する。

1. **読み戻し検証（決定的）**: `write` の戻り値の `verified` を見る。スクリプトが書き込み後にそのセルを読み戻し、`verified: true`（書いた内容と一致）/ `readback_chars`（実際に入った文字数）を返す。`verified: false` や文字数が大きく違うなら、改行潰れ・途中切れ・別セルへの誤爆を疑い、原因を直して書き直す（必要なら `--overwrite`）。`verified` が無い古い経路なら `read`/`gws sheets +read` で当該セルを読み戻して目視照合する。
2. **内容レビュー**: 読み戻した実セルに対して次を点検する。
   - テンプレ構成が揃っているか（`ハンズオン課題：` / `【詳細な要件】` / `【具体的な手順と学習ポイント】` / 最後にクリーンアップ）。
   - **無料枠・課金の注意とクリーンアップ手順**が入っているか（受講生の事故防止・最重要）。
   - AWS 手順が [[aws-docs]] 裏取りどおりで、参考リンクが実在するか（捏造が残っていないか）。
   - `filled_lessons` の既存課題と**主題が被っていない**か、直前レッスンから自然に積み上がっているか。
   - 途中で切れていない・想定の文字数に収まっているか（セル文字数上限は約 50,000 文字）。
3. **報告**: 上記レビュー結果（OK／要修正点）と `link`（`…#gid=…&range=セル`）を添えてユーザーに渡し、シート上でも確認してもらう。問題があれば握りつぶさず提示し、直す。

## 失敗時
- スクリプトの `ERROR:` はそのまま復旧手順になっている（未認証→ログインコマンド / セル埋まり→`--overwrite` / 権限403→アカウント確認 / gid不一致→存在タブ一覧）。出力を握りつぶさずユーザーに見せる。
