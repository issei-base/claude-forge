# 対象シートの構造とセル特定ロジック

`make-kadai` が前提にする SAMURAI ENGINEER カリキュラムシートの構造と、`sheet_kadai.py`
がどう対象セルを引き当てるか。挙動が想定と違う時にここを読む。

## インストラクター記入欄ブロック（課題が入る場所）

対象タブ（受講生ごとの記録タブ・URL の `gid` で指定）には、月割りで同じ構造のブロックが
1つ以上並ぶ。各ブロックは概ね次の並び:

```
列:    A (ラベル)            | B (1回目) | C (2回目) | … | 各レッスン列
------+---------------------+----------+----------+----+--------------
       1回目レッスン / 2回目… ← ヘッダ行（lesson→列 の対応元）
       インストラクター記入欄
       レッスンで指導した内容
       次回までの宿題          | <課題>   | <課題>   | … |  ← この skill が書く行
       次回レッスンの内容       | <予定>   | <予定>   | … |
       受講生記入欄
       宿題の進捗（必須）
```

実測した受講生タブの先頭ブロックではおよそ:
`次回までの宿題` = 行5 / `1回目レッスン` ヘッダから列、6回目 = 列 G。
（行番号・列は受講生シートごとに変わりうるので、上のラベル検出で都度引き当てる。）

## セル特定（`sheet_kadai.py` の決定的ロジック）

1. **lesson→列**: 全行を走査し「`N回目`」セルが最も多く並ぶ行をヘッダとみなす
   （`find_lesson_columns`）。全角数字「１回目」も hankaku 化して拾う。
2. **課題行**: 先頭付近のセルに「`次回までの宿題`」を含む**最初の**行（`find_kadai_row`）。
   月割りで複数ブロックがある場合は**先頭ブロック**を採る。
3. **対象セル**: `--lesson N` 指定時はその列、未指定時は**課題が空の最初のレッスン列**。
   既に埋まっている列には書かない（`write` は `--overwrite` 無しだとガードで止まる）。
4. **ヒント**: 「次回レッスンの内容」「レッスンで指導した内容」行から、直前回の値を拾って
   宿題の主題・接続のヒントにする（`resolve_target` の `hints`）。

別ブロックや特殊な配置に書きたい時は、`read` の出力でセルを確認して
`write --cell <A1>`（例 `--cell G15`）で明示する。

## 認証（書き込み）

- 読み: Sheets API GET（要 ADC スコープ）。※非公開シートでも編集権限アカウントなら読める。
- 書き: Sheets API `values.update`（`valueInputOption=USER_ENTERED` → URL は自動リンク化）。
- トークンは `gcloud auth application-default print-access-token`。スコープ不足/未認証は
  スクリプトが復旧コマンド付きで停止する。一度きりのセットアップは SKILL.md「前提」参照。
