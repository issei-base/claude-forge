# 投稿の型 — フック・構成・スケルトン

X でインプを取るための「型」集。**型 (フック・構成) は借りてよいが、中身は本物ネタで埋める**
(海外バズの丸コピは reach 激減 + 通報リスク)。以下はリサーチで裏取りした再利用可能なパターン。
末尾の出典に当たれる。

> ⚠️ エンゲージメント重み (返信 13.5×、ブックマーク 10× など) は **X の 2023 年公開アルゴリズム由来**で、
> 2026 年 1 月の再公開版は重みを伏せている。**順序の傾向 (返信 > リポスト/ブックマーク > いいね、
> 通報/ミュートは致命的) は堅いが、具体数値は確定値として書かない**こと。

## 1. フック・バンク (冒頭の型)

最初の 5〜7 語で読まれるか決まる。モバイルは最初の ~2 行しか見えない (「もっと見る」前)。
**(C)=Tips/ハマりどころ向き、(D)=逆張り・意見向き**。

| # | 型 | テンプレ | 向き |
|---|---|---|---|
| 1 | 逆張り / 神話崩し | `みんな [一般的なやり方] を勧める。でもそれ、[文脈] だと逆効果。` | D |
| 2 | Unpopular opinion | `たぶん不人気な意見: [反論可能だが筋の通った主張]` | D |
| 3 | Hard truth | `[AIコーディング] の不都合な真実: [居心地悪い現実]` | D |
| 4 | 誰も言わない gotcha | `誰も言わないけど、[症状] の原因は [非自明な理由] です。` | C |
| 5 | 数字付きリスト約束 | `[Claude Code] を爆速にする [N] 個のコツ:` (奇数が映える) | C |
| 6 | 「知っておきたかった」 | `[Claude Code] で [半年] 前に知りたかった [N] 個のこと 🧵` | C |
| 7 | Before/After 数字 | `[作業] が [3時間] → [12分] になった。ワークフロー全部出す:` | C |
| 8 | 「Y で X を作った」 | `[動くアプリ] を [Claude Code] で [40分] で作った。手書きコードはほぼゼロ。` | C |
| 9 | Stop/Start | `[手作業 X] をやめろ。[agent に任せろ]。週 [N] 時間浮く。` | C |
| 10 | 好奇心ギャップ | `[Claude Code] を大半が使いこなせてない。鍵は誰も触らない設定ひとつ:` | C/D |
| 11 | データで権威 | `[Claude] と [別ツール] を実タスク [50件] で比べた。結果は意外だった:` | C |

**選び方**: ホットテイク/gotcha は単発、ワークフロー/before-after/リストはスレ。
1投稿に詰め込みすぎない。1つの型 = 1つの投稿。

## 2. 構成: 単発 vs スレッド・文字数・余白

- **インプ最大化なら 3〜5 ツイートの短いスレが強い** (同テーマの単発 5 連投より総インプ +40〜60%、
  という調査あり・directional)。20 連投の大作スレは「終わった」。**3〜6 ツイートが今の最適**。
- **ホットテイク/即・gotcha は単発**が勝つ (返信を生む一撃)。
- **フックは最初の 2 行を ~100 字以内・自己完結**に。スレなら 1 ツイート目が単独で強いこと。
- **改行で 1 行 1 アイデア**。モバイルで読みやすく drop-off を減らす (JP でも 改行 は鉄則)。
- 絵文字はリスト記号として少量。盛らない。

## 3. メディア (画像/GIF/動画) の使いどころ

リサーチの一見矛盾を解くと:
- **reach (インプ) は 動画 > 画像 > テキスト**。9:16 縦・15〜60秒が有利。
- **エンゲージ率 (反応 ÷ インプ) は テキストが最高** (調査: text-only 3.24% > image 2.09% > video 1.34%)。
- → **インプ目的の最適は「テキスト先行フック + 実物スクショ/短い画面録画 GIF」**。
  ターミナル出力・diff・before/after・実際の CLAUDE.md など **"証拠ショット"** が最も拡散しやすい。
- claude-forge なら doc-illustrate の図解 HTML や 実 skill のスクショが格好の素材。

## 4. AI コーディングで刺さる角度

繰り返し当たる角度 (2025-2026):
- **「X を Y 分/$0 で作った」** — 2025 の支配的フォーマット。具体的で再現できそうに見える。
- **Before/After を数字で** — 「3時間 → 15分」。具体数字 = 信頼。
- **ワークフロー晒し** — 実ターミナル・CLAUDE.md・agent/MCP 設定のスクショ。
- **ツール比較 / bake-off** — 「Claude Code vs Cursor を実タスク N 件で」。引用RT を生む。
- **AI が dev を replace する/しない論** — 高アラウザル。返信は伸びるが ragebait 化を避け根拠を添える。控えめに。
- **「誰も言わない」gotcha** — 非自明な落とし穴・隠し設定・コスト/コンテキスト罠。ブックマーク + 返信に強い。
- **レシート/データ投稿** — 「N 件の PR/token を分析したら意外な事実が」。

## 5. 日本語 X 特有のチューニング

(JP マーケ情報源で裏取り・定性的。アルゴリズム自体は global で JP 専用ランキングは無い。)
- **文体 (ユーザー指定・最優先)**: バッククォート禁止 (X で ` がそのまま出る)。コマンド/パス/フラグは地の文に直書き。丁寧語 (です/ます) を使わず plain/casual 体。Tips bot ではなく「人が思ったことを書いた」声に — 本音・実感・軽い感情を一言。冒頭は問い・つぶやき・驚きで入ると自然。
- **レジスター (声の引き出し・実在の手本トーン)**。ネタに合わせて使い分ける:
  - **あるある自虐/ぼやき**: 「【悲報】Claude、Tool Call ミスりすぎて作業がひとつも進まない」。短く・共感ネタ・返信を生む。ネガっても重くせず笑いに寄せる。
  - **一人称の意見エッセイ**: 「僕は②の方が大きいと思っていて…」。長め・ニュアンス・断定しすぎない。番号で論点を並べる。
  - **オチのある実体験 Tips**: 「Codex に『有識者の意見』で渡すと鵜呑み、『インターンの意見』だと吟味してくれる。舐めてかかってる?笑」。具体的な発見 + 軽い笑い。保存もされる。
  - **ニュース + 自分の読み**: 「この論文を見るとやはり Mythos はサイバーセキュリティの能力がずば抜けて高いのがわかる」。事実を引いて自分の解釈を付ける。一次ソースは [[ ]] 不要だが出典は最初の返信へ。
  - **つぶやき/驚き**: 「すんげえ面白いの思いついた」「何も信じられない」。勢いで入る。
- **共感が JP の最大の拡散ドライバ**。EN の逆張り/権威より、体験談・共感を呼ぶ文章を中心に。
- **PREP (結論 → 理由 → 事例 → 結論)**。結論先出しが JP の規範。hook→proof→payoff と整合。
- **専門用語を絞る**。jargon は「分かる人だけに届く投稿」になり拡散を妨げる。フォロワー外に届けるなら日常語へ。
- **図解 (複数画像) が JP では特に効く** (dwell time)。+ 改行で可読性。
- **時間帯**: 早朝 5〜7時・通勤/夜が golden。混む枠を避ける手も。
- **インプ目安**: 1 投稿 = フォロワー数の 2〜3 倍が健康ライン。

## 6. スケルトン (構成テンプレ)

`[ ]` を本物ネタで埋める。**コピーするのは構造だけ**。

**A — 「X を Y で作った」単発 (インプ + ブックマーク)**
```
[成果物] を [Claude Code] で [所要時間] で作った。

手書きしたコードは [ほぼゼロ / N行]。

やったこと:
- [ステップ1]
- [ステップ2]
- [ステップ3]

[一行の学び / 意外だった点]
```
→ 実ターミナル/diff のスクショか GIF を添付。URL は最初の返信へ。

**B — 逆張り単発 (返信 + 引用RT)**
```
みんな [一般的なやり方] を勧める。

でもそれ、[文脈] だと逆効果。

理由は [簡潔な根拠]。

[代わりにこうする、を1〜2行]
```
→ メディア不要。テイクが本体。最初の 2 行 ~100 字以内。

**C — 「誰も言わない」gotcha (ブックマーク誘発)**
```
誰も言わないけど、[Claude Code] で [症状] になる原因はこれ:

[非自明な原因を1行で]

直し方:
1. [手順]
2. [手順]

[なぜ起きるか / 一行まとめ]
```
→ PREP で原因を結論先出し。保存されやすい。

**D — Before/After 数字スレ (3〜5 ツイート)**
```
1 (hook): [作業] が [旧 N時間] → [新 M分] になった。ワークフロー全部出す 🧵
2: 旧フロー: [手作業の内訳]。ボトルネックは [箇所]。
3: 新フロー: [Claude Code / agent の使い方を具体的に]。
4: [設定 / コマンド / CLAUDE.md の要点 — コピペできる形]
5: 結果: [数値で再掲]。[一行所感]。
```
→ 1 ツイート目は単独で強いフックに。各ツイート 1 アイデア・改行。

**E — ツール比較 / レシート (引用RT 誘発)**
```
[Claude Code] vs [別ツール] を実タスク [N件] で比べた。

結果は意外だった。

[A]: [強み / 弱み]
[B]: [強み / 弱み]

[条件付き結論: 「Xなら前者、Yなら後者」]

あなたはどっち派?
```

**F — 引用RT (quote-tweet) で話題に乗る (返信/引用RT 誘発)**

元ツイート (話題のニュース・他人の主張・バズ投稿) を引用しつつ、自分の take を一言乗せる。
配信は Typefully の quote_post_url で実現 (SKILL §5)。本体は「上に乗せるコメント」だけ書く。
```
[元ツイートへの一言の反応 / 同意・違和感・付け足し]

[自分の経験や具体で裏付け or 角度を足す 1〜2行]
```
- コメントは **80字以内が刺さる** (引用先カードで縦に伸びるので本文は短く)。
- 型: ①同意して具体を足す ②やんわり異論 (「わかる、でも〜」) ③自分の実体験で補強 ④ツッコミ/笑い。
- ネタ元の例: ユーザーが URL を渡す / フォロー中の List で流れてきた話題のツイート。
- **丸乗っかり (中身ゼロの『これ』だけ) は避ける**。必ず自分の角度を足す。Phase 1 では下書きにして人間が確認。
